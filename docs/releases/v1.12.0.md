# v1.12.0 — 2026-07-15

Agent-log analytics, secret scanning, and the ingestion advisor. Ships
roadmap Phases A (`agent_analytics`), C (`secret_scan`), and F
(`ingestion_advisor`) together in one release.

> Earlier drafts of the README referred to these three features under
> placeholder version numbers (1.8.1, 1.9.0). They actually shipped together,
> in this commit, as v1.12.0 — the README has since been corrected to match.

## Agent-log intelligence (`claude` lane)

- `claude_loop_check` — groups tool calls by session; reports repetition
  ratio, top repeated call, error-retry count, and a verdict (`healthy`,
  `some-repetition`, `likely-looping`).
- `claude_model_fit` — maps model names to a coarse tier (`frontier`, `mid`,
  `cheap`) and compares it to a complexity proxy built from tool count,
  errors, duration, and loop signals.
- `claude_token_burn` — uses `claude.tokens_input`/`claude.tokens_output`
  when present, falling back to `body characters / 4` per session.
  Computes burn per distinct tool plus inferred file target; every result
  labels its source as exact or estimated.
- `--agent-report` — runs all three checks headlessly and exits non-zero on
  a likely-looping or underpowered session, for cron/systemd alert piping.
  "high-burn" alone is excluded from the alert threshold since it is always
  present (a relative, top-decile ranking).

## Secret detection and output redaction

A stdlib-only secret scanner at the MCP output boundary, controlled by
`BERSERK_MCP_REDACT`:

- `redact` (default) — replaces detected values with typed placeholders
  (e.g. `[REDACTED:aws_key]`).
- `flag` — leaves the result intact, prepends a warning, logs a startup
  warning to stderr.
- `off` — disables scanning entirely, with a startup warning.

An unrecognized `BERSERK_MCP_REDACT` value fails closed to `redact`. The
scanner covers common cloud/provider credentials, private keys, JWTs,
bearer tokens, and generic password/token assignments; high-entropy
matching, email, IP, and Luhn-validated credit-card checks are each
individually selectable. `scan_secrets` audits recent log bodies but
returns only aggregate counts and timestamps, never matched values.

## Ingestion advisor

`suggest_ingestion` — an all-lane read-only tool backed by the editable
`ingestion_catalog.json`. Recommends concrete sources, explains why each
matters, names an ingestion mechanism, and labels maturity (`turnkey`,
`collector-receiver`, `bridge-required`, `manual`). Seeded use cases:
`sre/aws-cloud-native`, `sre/azure`, `sre/onprem-ad-health`,
`soc/endpoint-identity`, `change-management/ansible`, `scom`. With
`check_gap=true`, compares service/metric hints against the live Berserk
inventory and marks each `present` or `missing`.

## Review-pass fixes bundled in this release

- `scan_secrets` output no longer self-trips the global redaction filter.
- `--agent-report` no longer alerts on the always-present "high-burn" ranking.
- Token-usage attribute names are env-overridable
  (`BERSERK_MCP_TOKENS_IN/OUT_ATTR`).
- `claude_loop_check` scrubs secrets from echoed body snippets via an
  injected redactor, independent of the global filter mode.
- Programmatic-parsing modules request `--json` with graceful table
  fallback; parsers accept array/`{rows:[...]}`/jsonl/table shapes.
