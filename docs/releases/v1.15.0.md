# v1.15.0 — 2026-07-20

Phase J deep analytics, a multi-round security hardening pass (9 internal
findings + 3 external scanner findings), and the Discord alert integration
the README had already been describing.

## Phase J: deep analytics (`claude` lane)

- `claude_cost_report` — multi-day cost report: per-day burn with
  exact/estimated labels, per-model split, optional per-project attribution
  from file paths, and a burn-growing/flat/declining trend verdict.
- `claude_session_deep_dive` — one session's timeline: contiguous tool
  phases with error counts, activity gaps over 5 minutes, cumulative burn,
  and a loop verdict.
- `claude_workflow_insights` — cross-session patterns: most common tool
  sequences, error hotspots by tool+target, top-decile burn-per-target
  sessions.

Per-project cost attribution infers a project name from file-target paths
(the directory before the first marker segment — `src`, `tests`, `lib`,
`pkg` by default; override with `BERSERK_MCP_PROJECT_MARKERS`).

## Security hardening

- Multi-round internal security audit plus an external scanner pass.
- **SNYK-002 / CWE-23** — path-traversal guard for store-path env vars.
- **SNYK-003 / CWE-23** — path validation applied at every I/O sink.
- **F-001..F-005** — five findings closed from a Codex differential review.
- **F-006** — fixed the 500-item cap eviction to protect human-authored
  saved-query entries from being evicted ahead of generated ones.
- **F-007** — concurrency-safe writes across every JSON store.
- **F-008** — role visibility now enforced at `tools/call`, not just
  `tools/list` (a client could previously call a tool it couldn't see).
- **F-009** — `BERSERK_MCP_REDACT` now defaults to `redact` and fails
  closed on invalid config, rather than defaulting to a weaker mode.
