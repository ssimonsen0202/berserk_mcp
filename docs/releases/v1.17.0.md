# v1.17.0 — 2026-07-23

Adopts the native Berserk functions confirmed available in v1.16.0's KQL
performance guide across the query builders themselves.

Live-verified against the homelab cluster that `make-series`,
`series_fit_line`, `series_decompose_anomalies`, `series_fir`, `rate`,
`deriv`, `bin_auto`, `extract_log_template`, and `fieldstats` all work — see
the "Verified function availability" table in
[`docs/kql-performance-guide.md`](../kql-performance-guide.md).

## What changed

- `sre_error_rate`, `soc_log_spike` — `bin(timestamp, 1m)` + `sort` replaced
  with `make-series` (zero-filled buckets, no full-window sort).
- `soc_timeline`, `trace_find_errors`, `claude_recent`/`claude_errors`/
  `claude_search` — `sort` + `take` replaced with `tail`.
- `sre_top_errors`, `soc_repeated_errors` — now dedupe by
  `extract_log_template` instead of a raw body substring, keeping one
  representative example per template.
- `discover_schema` / parser-factory profiling — key discovery moved from
  `bag_keys` to `fieldstats` (type, cardinality, representative values);
  sample size reduced from 6 rows to 3.
- `claude_cost_report` — the trend verdict is now backed by a native
  `make-series` + `series_fit_line` query, reporting R², with the existing
  Python linear fit kept as the fallback for insufficient-data windows.

389/389 tests passing at release.
