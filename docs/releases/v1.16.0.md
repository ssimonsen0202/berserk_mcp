# v1.16.0 — 2026-07-23

Berserk-native query optimization pass, plus the per-project
cost-attribution fix that v1.17.0 later builds on.

## Query performance

- Dropped unneeded KQL sorts — Berserk's sort-before-take forces full-window
  materialization; `tail N` gets recency from the index instead.
- Narrowed every wide `body`/`resource`/`attributes` projection to what each
  consumer actually uses.
- Batched per-service drift probes into one grouped query.
- Cached `getschema` per profiling run.
- Where a consumer genuinely depends on input order
  (`analyze_workflow_events`, `analyze_model_fit_events`), restored it
  explicitly in Python with a locking test rather than relying on an
  implicit KQL order.

## Cost attribution

`_burn_events_query` now projects `claude.file_targets` (populated by the
matching `claude-otel-forwarder` change), with the old body-truncation
heuristic kept as a fallback for telemetry shipped before a forwarder
update.

## Docs

Added [`docs/kql-performance-guide.md`](../kql-performance-guide.md),
distilled from Berserk's own query-language docs (not Azure Data Explorer
assumptions) plus live verification against the homelab deployment.

388/388 tests passing at release (up from a 382 baseline plus 6 new
behavior-locking tests).
